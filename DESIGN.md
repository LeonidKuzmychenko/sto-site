# DESIGN.md

## 1. Назначение дизайна
Сайт — коммерческий лендинг-визитка для рекламы услуг.  
Основная аудитория — частные лица.  
Формат сайта фиксирован: **многостраничный лендинг**.

Допускается развитие:
- галереи с фотографиями
- видео-отзывы

Запрещено:
- превращение сайта в блог
- добавление информационных разделов без прямой коммерческой цели

---

## 2. Общая концепция
Тип дизайна: **минималистичный, сервисный, утилитарный**  
Приоритеты:
1. Читабельность
2. Чёткая визуальная иерархия
3. Предсказуемость интерфейса

Дизайн не должен отвлекать от услуги.

---

## 3. Цветовая система

### Базовая палитра
- Основной фон: `white`, `gray-50`
- Основной текст: `gray-900`
- Вторичный текст: `gray-600`
- Границы / разделители: `gray-200`

### Акцентные цвета
- Основной акцент: `blue`
- Дополнительные (только по смыслу):
    - `green` — успех, подтверждение
    - `red` — ошибка, предупреждение

### Ограничения
- Dark mode **не используется**
- Градиенты запрещены
- Цветные фоны секций — только при явной UX-необходимости
- Тени допустимы **только лёгкие и функциональные**

---

## 4. Типографика

### Шрифт
- По умолчанию: system-ui
- В будущем: возможен переход на Google Font (фиксируется отдельно)

### Иерархия
- Количество размеров текста на странице: **~4**
- Заголовки: `semibold / bold`
- Основной текст: `regular / medium`

### Правила
- `uppercase` допускается точечно (например навигация)
- Без декоративных шрифтов
- Без визуального «шума» за счёт типографики

---

## 5. Сетка и layout

- Контентный блок центрируется (`max-w-6xl`)
- Фон секций — full-width
- Основная структура: **строгая вертикальная колонка**
- Вертикальные отступы между секциями: `py-16`

Запрещено:
- асимметричные layout’ы
- «уникальные» секции, ломающие общий ритм

---

## 6. Навигация (Header)

- Header фиксирован вверху страницы
- При скролле остаётся на месте
- Имеет лёгкую прозрачность
- Использует `backdrop-blur`
- Разделён нижней границей

### Мобильная версия
- Burger-меню
- Overlay на весь экран

---

## 7. Компоненты

### Кнопки
- Основной тип: **solid**
- Цвет — акцентный (blue)
- Hover: изменение оттенка / прозрачности
- Без теней и псевдо-3D

### Карточки
- Прямоугольные
- Минимальные тени или border
- Чёткая иерархия: заголовок → текст → действие

### Списки
- С иконками
- Иконки не декоративные, а смысловые

---

## 8. Иконки
- Основной набор: **Lucide**
- При необходимости — кастомные SVG
- Запрещено смешивать разные стили иконок

---

## 9. Анимации и эффекты

### Разрешено
- Hover / focus эффекты
- Ненавязчивые UI-анимации
- Элегантные transition

### Запрещено
- Scroll-анимации секций
- Сложные keyframes
- Анимации без UX-смысла

Transition — стандартные значения Tailwind.

---

## 10. Адаптивность

- Подход: **desktop-first**
- Брейкпоинты — стандартные Tailwind
- На mobile допускается:
    - изменение структуры секций
    - скрытие второстепенного контента

Запрещено:
- дублирование контента ради адаптива

---

## 11. Контент и тон

- Тон: нейтральный, аккуратно продающий
- Текст короткий, визуально лёгкий
- Абзацы не перегружены

Допускается:
- восклицательные знаки
- маркетинговые клише

Запрещено:
- эмодзи
- «простыни» текста

---

## 12. Контроль качества дизайна

- Контроль соответствия стилю — по этому документу
- Если элемент не описан в DESIGN.md:
    - **документ обязан быть обновлён**
- Любые изменения дизайна → правка DESIGN.md

DESIGN.md — **живой документ**.

---

## 13. Anti-patterns (запрещено)

- Случайные цвета
- Уникальные стили «только здесь»
- Декоративные элементы без функции
- Смешение UI-подходов
- Анимации ради анимаций

---

## 14. Философия
Лучше сразу сделать хорошо, чем переделывать.  
Дизайн можно расширять, но нельзя размывать.

---

## 15. Frontend-стандарты разработки

### Архитектура
- Component-driven подход
- Один компонент — одна ответственность
- Логика отделена от JSX (через хуки)
- Переиспользование важнее уникальности

Запрещено:
- компоненты-комбайны
- логика внутри разметки
- скрытые зависимости

---

## 16. UI и Design System

- Tailwind — единственный источник визуальных решений
- Все размеры, цвета и отступы — только из design scale
- Вариативность через props / variants
- Любой новый визуальный паттерн требует обновления DESIGN.md

---

## 17. UX-принципы

- Один основной CTA на секцию
- Интерактив всегда очевиден
- Пользователь понимает:
    - где он
    - что можно сделать
    - что произойдёт дальше

Запрещено:
- скрытые действия
- неожиданные анимации
- клики без результата

---

## 18. SEO и семантика

- Semantic HTML обязателен
- Одна страница — один H1
- Логичная иерархия H2/H3
- Контент важнее декоративных элементов

Обязательно:
- alt у изображений
- lazy loading для медиа
- осмысленные тексты ссылок

Запрещено:
- SEO-текст без ценности
- div вместо семантических тегов
- контент, доступный только через JS

---

## 19. Контроль соответствия стандартам

- Любое отклонение от стандартов:
    - осознанное
    - зафиксированное
    - задокументированное
- DESIGN.md имеет приоритет над личными предпочтениями
